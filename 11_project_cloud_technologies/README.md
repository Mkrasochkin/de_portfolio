# Учебный проект по облачному хранилищу данных (Cloud Data Warehouse)

Проект описывает процесс построения простого облачного хранилища данных (DWH) с использованием микросервисной архитектуры и инструментов Yandex Cloud. Цель проекта — научиться основам ETL-процессов и автоматизации обработки данных из стриминговых источников.

## 👨‍💻 Описание проекта

Проект представляет собой учебную версию ETL-конвейера для обработки заказов. Система собирает данные из Kafka, нормализует их и выводит в виде аналитических витрин. Весь процесс построен на Kubernetes с использованием слоев STG (сырье), DDS (детали) и CDM (витрины).

## 🛠 Архитектура и рабочий процесс

1. **Staging Service**: Прием сырой информации из Kafka и помещение в PostgreSQL.
2. **DDS Service**: Распределение нормализованных данных по таблицам в слое DDS.
3. **CDM Service**: Генерация аналитических витрин на основе данных из DDS.
4. **DataLens**: Визуализация результатов на финальном этапе.

**Особенности**:
- Процедурный подход к нормализации данных.
- Управление жизненным циклом служб через K8s.
- Минималистичная логика обновления витрин.

## 🤝 Навыки и инструменты

- Язык программирования: Python.
- Библиотеки: Flask, APScheduler, psycopg2.
- Базовая работа с контейнерами: Docker, Kubernetes.
- Основы стриминга: Apache Kafka.
- Облачные инструменты: Yandex Cloud, DataLens.

## ✅ Общий вывод

Проект позволил освоить базовые концепции и инструменты дата-инженерии. Был пройден путь от приема данных из Kafka до их отображения в виде простых аналитических отчетов. Несмотря на академичность, освоены принципы работы с многослойными архитектурами и автоматизированной доставкой данных.
