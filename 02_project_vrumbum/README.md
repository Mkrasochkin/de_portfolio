# Автосалон «Врум-Бум»

## Описание проекта

Автосалон «Врум-Бум» столкнулся с ростом объемов данных о продажах, запасах и клиентах, усложняющим управление бизнесом. Вся эта информация сейчас хранится в сыром, неструктурированном виде, что сильно затрудняет работу (`cars.csv`). Цель проекта — нормализовать и структурировать существующие сырые данные, перенести их в PostgreSQL и реализовать ряд SQL-запросов для эффективного анализа. Проделанная работа позволяет руководству компании глубже понимать покупательские предпочтения, оптимизировать бизнес-процессы и повышать продажи, основываясь на качественной аналитике данных.

## Навыки и инструменты
PostgreSQL, DBeaver

- **PostgreSQL:** проектирование и создание схем данных, нормализация до 3-й нормальной формы, оптимизация структуры БД, написание SQL-запросов различной сложности.
- **DDL & DML:** создание и модификация таблиц, использование ограничений (constraints), нормализация данных, работа с автоинкрементами (SERIAL).
- **Join & Aggregation:** объединение таблиц различными методами (INNER, LEFT), группировка данных с использованием агрегатных функций (AVG, SUM, MAX, MIN).
- **ETL Processes:** разделение исходных данных на несколько взаимосвязанных таблиц, миграция данных из сырых данных в нормализованную структуру.
- **String Manipulation:** работа с текстовыми данными (функции SUBSTR, SPLIT_PART, CONCAT_WS), формирование сложных строковых выражений.
- **SQL Analytics:** расчет статистических показателей (средних значений, процентов, максимумов и минимумов), проведение аналитических расчетов с применением арифметических операций.
- **Index Optimization:** осознанный подбор типов данных и создание индексов для улучшения производительности запросов.
- **Date/Time Manipulation:** извлечение дат и времен из полей (YEAR, MONTH, DAY), группировка по периодам (месяца, годы).

## Общий вывод

В ходе выполнения проекта по созданию хранилища данных для автосалона «Врум-Бум» удалось разработать целостную, надежную и масштабируемую структуру данных, удовлетворяющую текущие и будущие потребности бизнеса. Была создана многослойная архитектура базы данных с применением принципа нормализации до 3-й нормальной формы, что повысило качество и быстроту обработки данных.

Были выполнены задачи:

- **Проектирование схемы данных:** созданы эффективные и взаимосвязанные таблицы, позволяющие легко осуществлять аналитику и поддерживать согласованность данных.
- **Нормализация данных:** проведена процедура нормализации до 3-й нормальной формы, что уменьшило избыточность данных и повысило производительность запросов.
- **Заполнение данными:** разработан ETL-процесс для миграции сырых данных из необработанного состояния в нормализованное хранилище.
- **Создание аналитических запросов:** выполнено несколько полезных аналитических запросов для расчета статистики, группировки данных и получения инсайтов для бизнеса.
