# Проект: Потоковая обработка данных для системы ситуативного маркетинга

## Описание проекта
Разработан сервис реального времени для агрегатора доставки еды. Основная задача — автоматизация отправки персонализированных push-уведомлений об акциях ресторанов пользователям, у которых данные заведения добавлены в список «Избранное». 

Сервис обеспечивает мгновенную обработку входящего потока маркетинговых кампаний из Kafka, сопоставляет их с базой подписчиков в PostgreSQL и формирует заготовки для уведомлений в период действия акции.

## Архитектура и поток данных

В проекте реализована архитектура с использованием **Spark Structured Streaming** в качестве ядра обработки.

```text
 [ Ресторан ] ──► [ Kafka ] ──► [ Spark Streaming ] ──► [ PostgreSQL ] ──► [ Kafka ] ──► [ Push Service ]
 (Акция)          (Raw JSON)    (Enrichment/Join)     (Feedback Log)      (Ready JSON)   (User Notification)
```
![Архитектура проекта](https://github.com/Mkrasochkin/de_portfolio/blob/main/10_project_stream_project/2._Proekt_1663599076.png) 


**Основные этапы работы сервиса:**
1.  **Ingestion (Прием)**: Чтение потока JSON-сообщений из топика Kafka через защищенное соединение (SASL_SSL).
2.  **Transformation (Трансформация)**: Десериализация данных из бинарного формата в типизированный датафрейм согласно JSON-схеме (`StructType`).
3.  **Validation (Валидация)**: Фильтрация акций «на лету» по времени актуальности (`start/end datetime`) относительно текущего UTC-времени системы.
4.  **Enrichment (Обогащение)**: Выполнение **Inner Join** потока акций с базой подписчиков из **PostgreSQL** по ключу `restaurant_id`.
5.  **Persistence & Egress (Выгрузка)**: Использование метода `foreachBatch` для одновременной записи обогащенных данных в аналитическую таблицу Postgres и отправки JSON-триггеров в выходной топик Kafka.



## Навыки и инструменты

### Потоковая обработка и Брокеры сообщений
*   **Apache Kafka**: Настройка взаимодействия между распределенными компонентами, работа с топиками, ключами сообщений и протоколами безопасности (SASL/SCRAM).
*   **Spark Structured Streaming**: Обработка данных в режиме реального времени, работа с микро-батчами и десериализация сложных JSON-структур.
*   **Performance Optimization**: Применение стратегий кэширования (`persist`/`unpersist`) для оптимизации ресурсов при записи данных в несколько целевых систем (Multi-sink).

### Инженерия данных и Интеграция
*   **Data Enrichment**: Навык объединения динамических потоков событий (Streams) со статическими справочниками из реляционных БД (JDBC Integration).
*   **Time-Series Analysis**: Работа с Unix Timestamp и системным временем для реализации логики временных окон и фильтрации событий «на лету».
*   **Testing & Debugging**: Использование утилиты **kcat (kafkacat)** для имитации входящих потоков и отладки логики обработки в реальном времени.



## Технические особенности реализации

*   **Идемпотентность и стабильность**: Обработка данных внутри микро-батчей позволяет гарантировать доставку сообщений и консистентность записей в PostgreSQL.
*   **Эффективное управление памятью**: За счет кэширования промежуточного результата Join-а в оперативной памяти исключаются избыточные повторные вычисления при одновременной записи в Kafka и БД.
*   **Гибкость схемы**: Использование строго заданных схем данных обеспечивает устойчивость сервиса к изменениям структуры входных сообщений.



## Общий вывод
В ходе реализации проекта построен высокопроизводительный сервис, объединяющий возможности событийной шины Kafka и реляционной базы данных. Удалось выстроить надежный процесс обогащения данных, позволяющий принимать интеллектуальные решения об отправке уведомлений с задержкой в несколько секунд.

Главным итогом стала автоматизированная система формирования персональных триггеров, учитывающая временные рамки маркетинговых активностей. Проект подтвердил компетенции в области построения **Real-time ETL** систем и управления ресурсами при распределенной обработке данных.
