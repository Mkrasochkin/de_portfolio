# Сервис уведомлений для агрегатора доставки (Kafka, Spark Streaming)

## Описание проекта
Разработан сервис потоковой обработки данных для агрегатора доставки еды. Задача — автоматизировать отправку уведомлений об акциях ресторанов тем пользователям, которые добавили эти заведения в список «Избранное». 
Сервис обеспечивает обработку входящего потока маркетинговых кампаний из Kafka, сопоставляет их с базой подписчиков в PostgreSQL и формирует заготовки для уведомлений в период действия акции.


## Архитектура и поток данных

В проекте реализована архитектура с использованием **Spark Structured Streaming** в качестве ядра обработки.

```text
 [ Ресторан ] ──► [ Kafka ] ──► [ Spark Streaming ] ──► [ PostgreSQL ] ──► [ Kafka ]
   (Акция)          (Raw JSON)    (Фильтр и Join)        (Feedback Log)    (Ready JSON)
```
![Архитектура проекта](https://github.com/Mkrasochkin/de_portfolio/blob/main/10_project_stream_project/2._Proekt_1663599076.png) 


**Основные этапы обработки:**
* **Чтение из Kafka**: Скрипт подключается к защищенному топику Kafka и забирает сообщения об акциях в формате JSON.
* **Десериализация**: Текст сообщения преобразуется в таблицу (DataFrame).
* **Фильтрация**: Проверка времени — акция обрабатывается только в том случае, если она актуальна в данный момент (текущее время попадает в интервал акции).
* **Объединение (Join)**: Поток акций соединяется со списком подписчиков, который вычитывается из базы PostgreSQL.
* **Сохранение**:
    * Данные отправляются обратно в Kafka (уже с ID конкретного пользователя) для сервиса уведомлений.
    * Копия данных записывается в PostgreSQL в таблицу обратной связи для будущего анализа.



## Навыки и инструменты

### Потоковая обработка (Spark и Kafka)
*   **Spark Structured Streaming**: Практика чтения данных микро-батчами. Работа с функциями `readStream` и `writeStream`.
*   **Интеграция с Kafka**: Настройка базового подключения к брокеру через защищенный протокол (SASL_SSL) и работа с JSON-сообщениями.
*   **Трансформация данных**: Преобразование типов данных (строки, числа, время) и фильтрация потока «на лету».

### Работа с базами данных
*   **PostgreSQL**: Чтение справочников через JDBC и запись результатов обработки обратно в базу.
*   **Обогащение данных**: JOIN динамического потока из Kafka со статичной таблицей из БД.

### Инженерные приемы
*   **Управление памятью**: Использование методов `persist` и `unpersist` для оптимизации ресурсов при записи данных в две разные системы одновременно.
*   **Тестирование**: Использование утилиты `kcat` для ручной отправки тестовых сообщений в Kafka и проверки работы кода.




## Общий вывод
В рамках проекта была реализована схема взаимодействия Spark Structured Streaming с внешними системами (Kafka и PostgreSQL). Основной задачей стала настройка процесса обогащения событийного потока статичными данными из реляционной базы, что позволило формировать уведомления с учетом интересов конкретных пользователей.
В ходе работы удалось отработать механизмы фильтрации данных «на лету» по временным меткам и реализовать логику записи одного микро-батча в несколько независимых приемников.
